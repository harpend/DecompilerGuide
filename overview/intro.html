What is a Decompiler?

The goal of a decompiler is to take executables and extract the code that generated them. This code is not available to us unless we go through the process of decompilation. Here are 2 examples:

C example

Java example

The reason I have chosen these 2 examples is because they are the 2 most common languages decompilers are written for, and they also represent 2 paradigms. Java compiles to bytecode, whereas C compiles to a binary file. This affects our process of decompilation since bytecode contains a lot of information not available in binary. Examples of things available in bytecode not available in binary include: Function names, parameter/return types, and segmentation of code/data.

Not having these features can significantly increase the complexity of a decompiler which is why it is considered easier to write a bytecode (Java) decompiler over one that decompiles from binary. There are however, some challenges universal to decompilers such as control flow reconstruction, data flow reconstruction and also obfuscation.
